# Composition Patterns And Originality Rules

## Basic Structure Types

Choose one structure type. Do not mix too many.

### Workflow

Best for: input -> processing -> output, content production, AI workflows, automation chains.

Drawing approach: input on the left, the Rough Rider operator or a strange machine processing in the middle, output on the right. Use orange arrows for the main flow.

### System Partial

Best for: information sources, filters, databases, renderers, local parts of an agent system.

Drawing approach: show only 3-5 core modules. The operator participates in one key action.

### Before-After

Best for: chaotic/ordered, manual/automatic, scattered/converged, anxious/stable.

Drawing approach: left side chaotic, right side stable, orange arrow in the middle. The operator can be exaggerated, but the system contrast should stay clear.

### Role State

Best for: user pain, creator state, too many tools, information anxiety, stuck-to-running transitions.

Drawing approach: 2-4 small states, each with one short English label.

### Conceptual Metaphor

Best for: content factory, information warehouse, mental black box, workflow machine, automated daily report.

Drawing approach: one large strange object or machine, a few inputs, one output. Make it memorable.

### Method Layers

Best for: methodology frameworks, system layers, capability stacks, content system layers.

Drawing approach: layered boxes, but not a formal pyramid. The operator is building, carrying, or repairing something nearby.

### Map Route

Best for: idea-to-launch, user path, content handoff path, learning route.

Drawing approach: one curved path, a few nodes, the operator walking, tying, or connecting the path.

### Mini Comic Sequence

Best for: failure-to-success, real process, critique, before/after change.

Drawing approach: 2-4 tiny scenes. Each panel expresses one action.

## Original Metaphor Method

Invent from the current article every time. Do not reuse old images.

### Three Steps

1. Turn the abstract concept into a physical action: stuck, leaked, heavy, sorted, settled, fermented, opened, folded, unpacked, returned.
2. Turn the system structure into a low-tech object: broken machine, cardboard box, drawer, pipe, mailbox, odd gauge, scale, well, ladder, strange workstation.
3. Make the operator perform the action: trapped in the machine, pulling the wrong wire, guarding a gate, carrying, repairing, weighing, holding a ladder, recording, or pushing something into a strange device.

### Object Pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, ball of string, gate, turntable, black box, hole punch, pasta press, clothesline, strange workstation.
- Use only 1-2 at a time. Do not fill the canvas.

### Operator Action Pool

- Pull, carry, stuff, fish out, press, weigh, sew, cut, twist, guard, push, catch, unpack, label, recycle.
- The action must serve the core idea. Do not be strange only for strangeness.

## Rough Rider Guardrail

Use Rough Rider influence for the recurring operator, not for the whole world. The surrounding composition should still feel like a clean product sketch with strange system objects. Add western props only when they genuinely clarify the article's metaphor.

## Anti-Copy Rules

Do not open or copy `assets/examples/` by default. These images are only for visual calibration: line density, whitespace, color restraint, and the operator's deadpan role.

Unless the user explicitly says "use this image", "copy this composition", or "remake this case", do not directly reuse these old compositions:

- Conveyor belt with two breakpoints.
- Operator pulling a judgment lever inside a content machine.
- Operator as a funnel sorting traffic/trust/conversion.
- Operator cutting a material fish.
- Operator pulling a handoff path.
- Operator pulling three-layer information sources.
- Three operators holding a megaphone, building a bridge, and opening a door.
- Operator stamping a copywriting toolbox.
- Operator holding a warning sign while looking at a common-pit path.

Even for similar themes, invent a new metaphor. For example, a "handoff path" does not have to be a route; it could be the operator attaching the tail of a content piece to a door handle. "One input, many uses" does not have to use a fish; it could show the operator pressing one paper ball into several shapes.
